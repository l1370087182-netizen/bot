# Binance Futures Quant Bot

一个面向 Binance U 本位合约的量化交易项目，包含：

- 实盘机器人
- Web 实时监控面板
- 中文 Excel 回测导出
- Walk-Forward 样本外验证
- 币种与方向分层策略

这个项目的目标不是做“回测很好看”的策略，而是尽量把规则策略做到更贴近未来实盘可执行的状态。

## 项目定位

当前版本已经从“只做规则尝试”走到了“有样本外验证、可进入模拟盘/小资金观察”的阶段。

核心原则：

- 优先实盘一致性，不优先回测报表
- 优先样本外验证，不优先全样本收益率
- 优先减少低质量交易，不优先盲目增加频次

## 当前策略概览

策略主框架：

- 主交易周期：`15m`
- 趋势确认周期：`1h`
- 市场环境过滤：`BTCUSDT 1h`
- 最大并发持仓：`2`
- 保本触发：`1.2R`
- 分批止盈：`2.0R / 25%`、`3.0R / 50%`
- 剩余仓位：ATR 追踪止损

核心信号组件：

- `ADX`
- `Stoch RSI`
- `EMA200`
- `Funding Rate`
- `CVD`
- 结构过滤
- 价格确认

这套策略不是“每个币一套完全独立策略”，而是：

- 一套统一主框架
- 币种分层
- 方向分层

## 当前币种分层

### 核心主池

- `BTC`
- `XRP`
- `AVAX`
- `SUI`

这些币在最新样本外结果里是主要正收益来源，更适合作为未来实盘重点观察对象。

### 方向限制池

- `ETH`：仅保留空头
- `SOL`：仅保留多头

原因不是“这个币不行”，而是它们在当前策略框架里，某一个方向明显不适配。

### 观察池

- `TRX`
- `BNB`

这两个币目前不是主要拖累，但也不是最强主力，适合继续观察，不适合当前就大幅放宽。

### 暂不作为重点

- `DOGE`
- `ADA`

这两个币在当前规则结构下噪音较大，已经通过方向分层大幅收缩了参与度。

## 为什么这个版本更接近实盘

这个仓库不是只保留“全样本最好看”的参数，而是按以下顺序推进：

1. 先做规则策略
2. 再做 Walk-Forward 样本外验证
3. 再做压力测试
4. 最后再决定是否进入模拟盘和小资金实盘

项目里已经明确做过的定向修正包括：

- 关闭 `ETH` 多头
- 关闭 `SOL` 空头
- 关闭 `DOGE` 多头
- 关闭 `ADA` 多头
- 对 `BTC / XRP / AVAX / SUI` 做更积极但仍有限制的强币分层

这类调整的目标不是“让回测更漂亮”，而是减少明显不适配的方向，让未来实盘更稳定。

## 主要文件说明

### 交易与策略

- `bot.py`
  实盘机器人主程序，负责交易所连接、K 线监听、信号执行、持仓管理

- `strategy.py`
  策略核心逻辑，包括指标计算、信号判断、评分、结构过滤、止盈止损和风控

- `config.py`
  所有核心参数、币种分层、方向限制、风控和代理配置

### 回测与验证

- `backtest_to_excel.py`
  全样本离线回测，并导出中文 Excel 报告

- `walk_forward_backtest.py`
  Walk-Forward 样本外验证与压力测试框架

### Web 面板

- `web_server.py`
  Web 后端服务

- `web_dashboard.html`
  Web 仪表盘页面

### 文档

- `docs/PROJECT_SUMMARY.md`
  当前策略、结果、风险和下一步建议的项目摘要

- `AGENTS.md`
  本项目的持久化协作指令

## 环境要求

- Windows
- Python 3.10+
- 可访问 Binance 的 HTTP / WebSocket 代理

## 安装

```bash
pip install -r requirements.txt
```

## 配置

1. 复制环境变量模板

```bash
copy .env.example .env
```

2. 填写你自己的 Binance API Key / Secret

3. 如需代理，配置：

```env
PROXY_ENABLED=true
PROXY_URL=http://127.0.0.1:7897
```

## 运行方式

### 启动 Web 面板

```bash
python web_server.py
```

默认地址：

```text
http://127.0.0.1:8081
```

### 启动机器人

实盘：

```bash
python bot.py --real
```

测试网：

```bash
python bot.py --real --testnet
```


### Walk-Forward 样本外验证

## 当前建议使用方式

推荐顺序：

1. 先模拟盘运行一周
2. 重点观察信号触发、方向限制、日志、代理、持仓和风控是否稳定
3. 模拟盘稳定后，再进入小资金实盘
4. 在规则策略足够稳定之前，不急着引入 ML/AI

## 关于 ML / AI

这个项目后续可以接 ML / AI，但当前建议：

- 先把规则策略做到满意
- 再让 ML 做增强层

更适合的方向是：

- 信号质量过滤
- 市场状态分类
- 动态风险分层
- 退出增强器

暂时不建议让 ML 直接替代当前主策略去决定开仓和平仓。

## 注意事项

- 不要提交 `.env`、日志、数据库和缓存文件
- Binance Futures 对账户类 REST 请求有限频，优先走 WebSocket 与缓存
- 代理出口稳定性会直接影响实盘质量
- 样本外比全样本更接近实盘，但依然不等于真实实盘
