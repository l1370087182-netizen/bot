## 上下文管理规则

- 每次响应前，先检查当前背景信息窗口占用率（Background Information / Project Context）。
- 如果占用率 >= 75%（或接近 80%），立即执行以下操作：
  1. 对当前项目核心信息做一次深度总结。
  2. 生成或更新 Markdown 文件：`docs/PROJECT_SUMMARY.md`
  3. 文件必须包含：策略核心逻辑、回测框架、最新参数表、绩效指标、已知风险、当前日期。
  4. 自动创建并保存该文件。
  5. 回复中明确说明：“已自动保存 PROJECT_SUMMARY.md（上下文已优化）”。

- 完成后继续正常响应用户请求。

此规则永久生效于本项目所有会话。
